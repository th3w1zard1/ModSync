# Documentation hierarchy

`[REPO]` Authority order for conflicting guidance. Prefer the highest row unless the user explicitly overrides.

| Priority | Document | Role | Edit when… |
|----------|----------|------|------------|
| 1 | User message / issue / PR description | Task-specific intent | N/A |
| 2 | `.cursorrules` | Always-on constraints (paths, VFS, tests, GUI rules) | Changing sandbox or test policy |
| 3 | `AGENTS.md` | Repo-root routing for agents (GUI vs headless vs sidecar) | Wizard map, preload args, releases pointer |
| 4 | `.github/copilot-instructions.md` | Copilot-first defaults and build commands | Autonomous inference or command changes |
| 5 | `docs/knowledgebase/` | Canonical index, audits, parity maps | New agent surfaces or major workflow changes |
| 6 | `docs/local_desktop_agent_runbook.md` | Deep GUI/full-build procedures | Verified desktop workflow changes |
| 7 | `docs/manual-release.md` | Release publish runbook | CI/release workflow changes |
| 8 | `docs/solutions/` | Institutional learnings | After shipping a non-obvious fix |
| 9 | `docs/plans/` | Dated implementation plans | Each planned slice of work |
| 10 | `.cursor/skills/*` | Cursor skill shortcuts | When skills drift from rows 2–7 |
| 11 | Generated docs (`KOTORModSync_Master.md`, etc.) | Reference dumps | Regenerated by tooling, not hand-edited |

## Layering model

```text
Always-on trio (.cursorrules + AGENTS.md + copilot-instructions)
        │
        ▼
Knowledgebase index (docs/knowledgebase/README.md)
        │
        ├── Runbooks (how to execute)
        ├── Solutions (why we chose X)
        ├── Plans (what we intended to ship)
        └── Scripts catalog (automation entry points)
```

## Drift prevention

When updating agent guidance:

1. Change the **source-of-truth trio** first if the rule is always-on.
2. Update **knowledgebase** links or topic pages if discoverability changes.
3. Update **runbooks/skills** only for procedural steps that changed.
4. Add a **solution** doc when the learning should survive beyond one PR.

Do not copy long procedural sections into `AGENTS.md` — link instead.
